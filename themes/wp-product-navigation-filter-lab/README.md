## 项目简介

本项目是一套面向 WordPress 经典主题开发的产品分类与筛选学习主题。

主题将以下技术组合成一套可以直接安装、运行、调试和继续扩展的完整案例：

- `product` 产品自定义文章类型
- `product_category` 多层级产品分类
- 品牌、电压、功能特点分类法
- 产品型号、价格和功率字段
- 多层级产品分类导航
- 产品分类面包屑
- 返回上一级分类
- GET 多条件筛选
- 已选择与已应用状态区分
- 单项、单组和全部条件清除
- `pre_get_posts`
- `tax_query`
- `meta_query`
- 筛选参数白名单验证
- 产品排序
- 筛选条件分页保留
- 产品后台管理列与数值排序
- 演示数据一键导入

当前版本：

```text
v1.1.0
```

## 安装与升级

### 新安装

1. 进入 `外观 → 主题 → 安装主题 → 上传主题`；
2. 上传主题 ZIP 压缩包；
3. 启用“产品导航筛选实验室”；
4. 进入 `外观 → 产品主题演示数据`；
5. 点击“一键导入演示数据”；
6. 进入 `设置 → 固定链接`；
7. 点击一次“保存更改”；
8. 打开 `/products/`。

### 从 v1.0.0 升级

主题目录名称保持不变，可以覆盖安装或替换主题文件。

升级不会删除已经创建的产品、分类和产品属性。升级后建议：

1. 清除浏览器和网站缓存；
2. 进入固定链接页面保存一次；
3. 测试产品归档、分类导航、筛选、排序和分页；
4. 检查后台“全部产品”的新增管理列。

## 多层级分类导航原理

核心函数：

```php
pfl_get_product_category_index()
pfl_get_current_product_category_path()
pfl_get_category_navigation_levels()
pfl_product_category_has_children()
```

v1.1.0 会一次获取全部 `product_category` 分类，并建立：

```php
[
    'by_id'     => [],
    'by_parent' => [],
]
```

之后在 PHP 数组中完成：

- 一级分类查询；
- 当前路径恢复；
- 直接子分类读取；
- 是否存在下级分类判断；
- 面包屑生成。

这样可以避免在分类循环中不断调用 `get_terms()`。

## 分类路径样式

分类导航区分三种状态：

```text
祖先分类：浅蓝色
当前分类：深蓝色
下一层可选分类：白色
```

例如访问：

```text
工业设备 → 包装设备 → 真空包装机
```

页面会同时显示一级、二级和三级导航，并明确标记当前所在分类。

---

## 筛选状态原理

v1.1.0 区分两个状态：

```text
已选择条件
已经应用条件
```

用户点击条件后，前端会比较当前表单状态和页面初始状态。

尚未提交时显示：

```text
已选择 2 个条件，尚未应用
```

提交并重新加载后显示：

```text
当前已应用 2 个筛选条件
```

这可以避免按钮已经高亮、但产品结果尚未更新时产生误解。

## 筛选配置

分类法筛选集中在：

```php
pfl_get_taxonomy_filter_config()
```

范围筛选集中在：

```php
pfl_get_range_filter_config()
```

排序集中在：

```php
pfl_get_sort_options()
```

新增筛选项时，应优先修改配置，而不是在多个模板和查询函数中重复增加判断。

---

## 参数白名单验证

主题会对 URL 参数执行两层处理：

1. 使用 WordPress sanitization 函数清理输入；
2. 检查 term slug 或范围值是否存在于允许列表中。

分类法筛选使用：

```php
pfl_get_allowed_taxonomy_slugs()
```

非法品牌、电压、功能、价格范围、功率范围和排序参数会被忽略。

---

## 主查询修改

主题继续使用：

```php
add_action(
    'pre_get_posts',
    'pfl_filter_product_main_query'
);
```

模板中不重新创建第二个 `WP_Query`。

筛选关系：

```text
品牌：IN
电压：IN
功能特点：AND
不同筛选组：AND
```

数值范围使用：

```php
meta_query
```

排序支持：

```text
最新发布
价格从低到高
价格从高到低
标题排序
```

## 提交后定位结果

筛选表单 action 带有：

```text
#product-results
```

因此提交并重新加载页面后，会自动定位到产品结果区域，减少重复滚动。

---

## 无结果状态

筛选结果为 0 时，页面会根据是否存在筛选条件显示不同信息。

存在筛选条件时提供：

- 清空当前筛选；
- 查看全部产品。

没有筛选条件时提示用户添加产品或切换分类。
